# Kanban Overview

This kanban board system organizes tasks into three states:

- **to-do**: Tasks that are planned but not yet started
- **in-progress**: Tasks currently being worked on
- **done**: Completed tasks

## Task Numbering

Tasks use a three-digit numbering system (001-999).

Example: `001-implement-component-base.md`

## Task Lifecycle

1. Create task in `to-do/` using the task template
2. Move to `in-progress/` when starting work
3. Update checklist items as you progress
4. Move to `done/` when completed
5. Add results/notes section documenting outcomes

## File Naming

Use kebab-case with three-digit prefix: `NNN-task-description.md`
